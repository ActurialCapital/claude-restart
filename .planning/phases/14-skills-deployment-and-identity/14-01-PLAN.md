Add a `deploy_skills()` function to bin/install.sh that deploys GSD and superpowers skills to ~/.claude/ on the VPS. Place it after the `migrate_v1_env()` function and before `do_install_linux()`.

The function should:

```bash
deploy_skills() {
    local skills_src="$SCRIPT_DIR/../skills"
    local commands_src="$SCRIPT_DIR/../commands"
    local claude_dir="$HOME/.claude"

    # Deploy GSD skills (get-shit-done)
    if [[ -d "$skills_src/get-shit-done" ]]; then
        mkdir -p "$claude_dir/get-shit-done"
        cp -r "$skills_src/get-shit-done/"* "$claude_dir/get-shit-done/"
        echo "Deployed GSD skills to $claude_dir/get-shit-done/"
    else
        echo "Warning: GSD skills not found at $skills_src/get-shit-done (skipping)"
    fi

    # Deploy superpowers commands
    if [[ -d "$commands_src" ]]; then
        mkdir -p "$claude_dir/commands"
        cp -r "$commands_src/"* "$claude_dir/commands/"
        echo "Deployed superpowers commands to $claude_dir/commands/"
    else
        echo "Warning: superpowers commands not found at $commands_src (skipping)"
    fi
}
```

Then add a call to `deploy_skills` inside `do_install_linux()`, after the env.template copy (step 1b, around line 97) and before the v1.1 env migration. Add it as step "1c" with a comment:

```bash
    # 1c. Deploy skills (GSD + superpowers) to ~/.claude/ (Phase 14)
    deploy_skills
```

Renumber the existing "1c. Migrate v1.1 env" comment to "1d".

Per D-01 (Claude's discretion on deployment mechanism): Using direct cp from repo source during install. The repo will need skills/ and commands/ directories bundled. Per D-02 (update sync strategy): Manual re-run of install.sh to update skills — no auto-sync. Per D-05 (deployment automation): Folded into existing install.sh flow.

Note: The actual skills source directories (skills/get-shit-done/ and commands/) will be created as placeholder/README directories in the repo in Task 2, since the real skills live at ~/.claude/ on the dev machine and are not part of this repo. The install.sh deploy_skills function gracefully skips if directories are missing (warning message).
