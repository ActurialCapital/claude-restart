<objective>
Add instrument identity CLAUDE.md injection to claude-service and fix session naming for phone display.

Purpose: Instruments on VPS currently have no awareness of their own instance name. This plan creates a per-instrument CLAUDE.md identity file deployed to .claude/CLAUDE.md (project-level, does NOT overwrite repo root CLAUDE.md) during `claude-service add`. This gives each instrument self-awareness and provides the `claude-restart --instance <name>` hint. The session fix is addressed by the existing --name flag in claude-wrapper (already passes CLAUDE_INSTANCE_NAME to remote-control --name), which names sessions distinctly — the fix ensures this works correctly for all instruments including orchestra.

Output: instrument-CLAUDE.md.template, updated claude-service with identity deployment, updated tests.
</objective>

**Part A: Create instrument-CLAUDE.md.template at repo root.**

This template gets deployed to `~/instruments/<name>/.claude/CLAUDE.md` (NOT the repo root CLAUDE.md — the .claude/ subdirectory is a project-level Claude config location that Claude Code reads without overwriting the repo's own CLAUDE.md). Per D-03 (Claude's discretion on identity injection).

Content:
```markdown
# Instrument: INSTANCE_PLACEHOLDER

You are the **INSTANCE_PLACEHOLDER** instrument running on a VPS. You are a Claude Code session managed by systemd, accessed via remote-control or autonomous dispatch.

## Identity

- **Instance name:** INSTANCE_PLACEHOLDER
- **Working directory:** ~/instruments/INSTANCE_PLACEHOLDER
- **Config:** ~/.config/claude-restart/INSTANCE_PLACEHOLDER/env

## Context Reset

If you need a fresh context window (e.g., after a long GSD session), tell the user or orchestra to run:

```
claude-restart --instance INSTANCE_PLACEHOLDER
```

This kills your current process and systemd restarts you with a clean slate. Your project files and .planning/ state persist — only your conversation context resets.

## Remote Access

The user can connect to you directly via:

```
claude remote-control --name INSTANCE_PLACEHOLDER
```
```

**Part B: Update do_add() in bin/claude-service to deploy identity CLAUDE.md.**

After the git clone step (line ~80, `git clone "$git_url" "$work_dir"`) and before the env creation, add:

```bash
    # Deploy instrument identity CLAUDE.md (Phase 14, per D-03)
    local claude_md_template
    claude_md_template="$script_dir/../instrument-CLAUDE.md.template"
    if [[ -f "$claude_md_template" ]]; then
        mkdir -p "$work_dir/.claude"
        cp "$claude_md_template" "$work_dir/.claude/CLAUDE.md"
        sed_inplace "s|INSTANCE_PLACEHOLDER|$name|g" "$work_dir/.claude/CLAUDE.md"
        echo "Deployed instrument identity to $work_dir/.claude/CLAUDE.md"
    fi
```

Note: do_add() currently does not define `script_dir`. Add it near the top of do_add(), after the local declarations:
```bash
    local script_dir

```

**Part C: Fix session naming for default instance (SESS-01).**

The duplicate "General coding session" on phone happens because the default instance skips `--name` in claude-wrapper (line 24: `"${CLAUDE_INSTANCE_NAME}" != "default"` check). Fix this in bin/claude-wrapper by removing the "default" exclusion. Change:

```bash
        if [[ -n "${CLAUDE_INSTANCE_NAME:-}" && "${CLAUDE_INSTANCE_NAME}" != "default" ]]; then
            mode_args+=("--name" "$CLAUDE_INSTANCE_NAME")
        fi
```

To:

```bash
        if [[ -n "${CLAUDE_INSTANCE_NAME:-}" ]]; then
            mode_args+=("--name" "$CLAUDE_INSTANCE_NAME")
        fi
```

This ensures even the "default" instance gets `--name default` which gives it a distinct session name on phone instead of "General coding session".
