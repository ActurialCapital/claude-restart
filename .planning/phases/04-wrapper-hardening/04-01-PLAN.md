    **Signal handling changes to bin/claude-wrapper (per D-01, D-02, D-03):**

    1. Replace the current `trap 'exit 130' INT` on line 14 with three traps:
       - `child_pid=""` variable initialized before the loop
       - SIGTERM trap: `trap 'if [[ -n "$child_pid" ]]; then kill -TERM "$child_pid" 2>/dev/null; wait "$child_pid" 2>/dev/null; fi; exit 0' TERM`
       - SIGHUP trap: `trap '' HUP` (ignore — empty string handler)
       - SIGINT trap stays: `trap 'exit 130' INT`

    2. Change `claude "${current_args[@]}"` (line 18) to run claude in background and wait:
       ```
       claude "${current_args[@]}" &
       child_pid=$!
       wait "$child_pid" 2>/dev/null
       exit_code=$?
       child_pid=""
       ```
       Note: `wait` returns the child's exit code. If interrupted by a trapped signal, wait returns 128+signal. After wait completes normally, clear child_pid so the trap doesn't try to kill a dead process.

    3. Handle wait being interrupted by signal: when a signal arrives during `wait`, bash interrupts it and runs the trap handler. The trap handler for TERM does `wait "$child_pid"` again to get the actual exit code. After the trap runs, the main `wait` returns with 128+signum. Add a check: if exit_code > 128, it means a signal interrupted wait — the trap already handled cleanup, so just continue the loop (the TERM trap exits, so this path only matters for other signals).

    **Mode selection changes to bin/claude-wrapper (per D-04, D-05, D-06, D-07, D-08):**

    4. After the `original_args=("$@")` line (line 10), add mode resolution block:
       ```bash
       # Mode selection (per CLAUDE_CONNECT env var)
       mode_args=()
       case "${CLAUDE_CONNECT:-}" in
           remote-control)
               mode_args=("remote-control")
               ;;
           telegram)
               mode_args=("--channels" "plugin:telegram@claude-plugins-official")
               ;;
           "")
               # Interactive mode — no mode args, use passed args only
               ;;
           *)
               echo "claude-wrapper: unknown CLAUDE_CONNECT mode: $CLAUDE_CONNECT" >&2
               echo "claude-wrapper: valid values: remote-control, telegram, or unset for interactive" >&2
               exit 1
               ;;
       esac
       ```

    5. Change the claude launch line to prepend mode_args:
       ```bash
       claude "${mode_args[@]}" "${current_args[@]}" &
       ```
       Use `"${mode_args[@]}"` which expands to nothing when the array is empty (interactive mode).

    6. Similarly, in the restart branch (around line 39-47), ensure mode_args are always prepended when relaunching. The echo messages should show the full args including mode: `echo "Restarting claude with: ${mode_args[*]} ${current_args[*]}"`

       Run wrapper in background, read child PID from file, send SIGTERM to wrapper PID, wait briefly, check signal log file contains "GOT_SIGTERM", check wrapper exited 0.

    8. Add test for SIGHUP ignore. Run wrapper with a mock that sleeps, send SIGHUP to wrapper, verify wrapper is still running (kill -0), then clean up.

    9. Add tests for CLAUDE_CONNECT modes:
       - Set `CLAUDE_CONNECT=remote-control`, run wrapper with `--verbose`, check mock log first line is `remote-control --verbose`
       - Set `CLAUDE_CONNECT=telegram`, run wrapper, check mock log contains `--channels plugin:telegram@claude-plugins-official`
       - Unset CLAUDE_CONNECT, run wrapper with `--foo`, check mock log is `--foo` (backwards compat)
       - Set `CLAUDE_CONNECT=bogus`, run wrapper, check exit code is 1, check stderr contains "unknown CLAUDE_CONNECT"

    10. Add test for mode + extra args: `CLAUDE_CONNECT=remote-control` with args `--verbose --model opus`, verify logged args are `remote-control --verbose --model opus`.
