Modify bin/claude-wrapper to fix two issues in remote-control mode:

**Fix 1: Permission flag**
In the remote-control case of the mode selection switch (lines 22-26), add `--permission-mode bypassPermissions` to mode_args:

```bash
remote-control)
    mode_args=("remote-control" "--permission-mode" "bypassPermissions")
    if [[ -n "${CLAUDE_INSTANCE_NAME:-}" && "${CLAUDE_INSTANCE_NAME}" != "default" ]]; then
        mode_args+=("--name" "$CLAUDE_INSTANCE_NAME")
    fi
    ;;
```

Additionally, in the remote-control mode, any `--dangerously-skip-permissions` that appears in current_args must be filtered out since it is invalid for the remote-control subcommand. Add a filter block after the mode selection switch (after line 39, before the channel_args block):

```bash
# Filter --dangerously-skip-permissions from current_args in remote-control mode
# (remote-control uses --permission-mode bypassPermissions instead)
if [[ "${CLAUDE_CONNECT:-}" == "remote-control" ]]; then
    filtered_args=()
    for arg in "${current_args[@]}"; do
        if [[ "$arg" != "--dangerously-skip-permissions" ]]; then
            filtered_args+=("$arg")
        fi
    done
    current_args=("${filtered_args[@]}")
fi
```

**Fix 2: Auto-confirm Enable Remote Control prompt**
In the main loop, for remote-control mode specifically, pipe 'y' followed by a newline to stdin so the "Enable Remote Control? (y/n)" prompt is auto-accepted. Modify the else branch at line 92-94 to handle remote-control separately:

```bash
    elif [[ "${CLAUDE_CONNECT:-}" == "remote-control" ]]; then
        # Auto-confirm "Enable Remote Control?" prompt for non-interactive service startup
        echo "y" | claude "${mode_args[@]}" "${channel_args[@]}" "${current_args[@]}" &
        child_pid=$!
    else
        claude "${mode_args[@]}" "${channel_args[@]}" "${current_args[@]}" &
        child_pid=$!
    fi
```

This changes the if/elif/else structure from:
- if telegram: launch with FIFO
- else: launch normally

To:
- if telegram: launch with FIFO
- elif remote-control: launch with `echo "y" |` for auto-confirm
- else: launch normally (interactive)

Do NOT change any other behavior: telegram heartbeat, restart logic, signal handling, channel_args, or the original_args/current_args restart mechanism must remain identical.
