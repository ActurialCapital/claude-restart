Fix remote-control mode to use FIFO-based stdin with heartbeat writer instead of `echo "y" |` pipe.

Purpose: The `echo "y" | claude ...` pattern on line 106 causes three fatal issues: (1) EOF on stdin kills the remote-control process, (2) "y" leaks into the session instead of the confirmation prompt, (3) the session is one-shot with no way to receive further input. The telegram mode (lines 82-103) already solves this correctly with a FIFO + heartbeat writer pattern. Remote-control mode needs the same pattern.

Output: `bin/claude-wrapper` with FIFO stdin for remote-control, updated tests

   - Open fd 3 to the FIFO (`exec 3>"$HEARTBEAT_FIFO"`)
