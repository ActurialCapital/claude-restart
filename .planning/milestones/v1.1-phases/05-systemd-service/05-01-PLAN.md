Create `systemd/claude.service` with this exact content:

```ini
[Unit]
Description=Claude Code AI Assistant
After=network-online.target
Wants=network-online.target
StartLimitBurst=5
StartLimitIntervalSec=60

[Service]
Type=simple
ExecStart=%h/.local/bin/claude-wrapper --dangerously-skip-permissions
EnvironmentFile=%h/.config/claude-restart/env
WorkingDirectory=WORKING_DIR_PLACEHOLDER
Restart=on-failure
RestartSec=5
KillSignal=SIGTERM
TimeoutStopSec=10

[Install]
WantedBy=default.target
```

Key design notes per locked decisions:
- `StartLimitBurst=5` and `StartLimitIntervalSec=60` in `[Unit]` section, NOT `[Service]` (per D-05)
- `Restart=on-failure` not `always` (per D-04) to avoid double-restart with wrapper
- `--dangerously-skip-permissions` hardcoded in ExecStart (per D-03)
- `EnvironmentFile` points to env file (per D-02) — no Environment= lines
- `WorkingDirectory=WORKING_DIR_PLACEHOLDER` — installer replaces this (per D-06)
- `%h` expands to user home directory in systemd user units
- `RestartSec=5` (Claude's discretion — reasonable delay between crash and restart)
- `KillSignal=SIGTERM` ensures wrapper's TERM trap fires on `systemctl stop`
- `TimeoutStopSec=10` gives wrapper time to forward SIGTERM to claude child

Create `bin/claude-service` with this exact content:

```bash
#!/bin/bash
# claude-service: manage the claude systemd user service
set -euo pipefail

SERVICE="claude.service"

usage() {
    echo "Usage: claude-service {start|stop|restart|status|logs}"
    echo ""
    echo "Commands:"
    echo "  start    Start the claude service"
    echo "  stop     Stop the claude service"
    echo "  restart  Restart the claude service"
    echo "  status   Show service status"
    echo "  logs     Follow service logs (journalctl)"
}

case "${1:-}" in
    start)
        systemctl --user start "$SERVICE"
        echo "Claude service started"
        ;;
    stop)
        systemctl --user stop "$SERVICE"
        echo "Claude service stopped"
        ;;
    restart)
        systemctl --user restart "$SERVICE"
        echo "Claude service restarted"
        ;;
    status)
        systemctl --user status "$SERVICE"
        ;;
    logs)
        journalctl --user -u claude -f
        ;;
    --help|-h|"")
        usage
        ;;
    *)
        echo "Unknown command: $1" >&2
        usage >&2
        exit 1
        ;;
esac
```

Make the file executable: `chmod +x bin/claude-service`

Per D-12: exactly 5 subcommands (start, stop, restart, status, logs).
Per D-13: `logs` wraps `journalctl --user -u claude -f`.
Per D-14: no separate log file — journald only.
