Modify `bin/claude-wrapper` to add a heartbeat background process that sends empty input to claude's stdin every 5 minutes, but ONLY when `CLAUDE_CONNECT=telegram`.

Per D-04: Send periodic newline to claude's stdin every 5 minutes in telegram mode.
Per D-05: Each heartbeat logs a timestamp to stderr (which journald captures).
Per D-07: Hardcoded 5 minutes, not configurable in v1.1.
Per D-11: Heartbeat gated on `CLAUDE_CONNECT=telegram` — no heartbeat in remote-control or interactive mode.
Per D-16: Heartbeat is built into claude-wrapper, no separate service.

Implementation approach — use a background subshell that writes to a named pipe (FIFO) or directly to the process's stdin fd. The cleanest approach given the existing `wait` pattern:

1. Add a `heartbeat_pid=""` variable at the top alongside `child_pid=""`.

2. Create a `start_heartbeat()` function that launches a background loop:
```bash
start_heartbeat() {
    if [[ "${CLAUDE_CONNECT:-}" != "telegram" ]]; then
        return
    fi
    (
        while true; do
            sleep 300
            if [[ -n "$child_pid" ]] && kill -0 "$child_pid" 2>/dev/null; then
                echo "" > /proc/$child_pid/fd/0 2>/dev/null || true
                echo "claude-wrapper: heartbeat sent $(date -u +%Y-%m-%dT%H:%M:%SZ)" >&2
            else
                break
            fi
        done
    ) &
    heartbeat_pid=$!
}
```

3. Create a `stop_heartbeat()` function:
```bash
stop_heartbeat() {
    if [[ -n "$heartbeat_pid" ]]; then
        kill "$heartbeat_pid" 2>/dev/null
        wait "$heartbeat_pid" 2>/dev/null || true
        heartbeat_pid=""
    fi
}
```

4. Call `start_heartbeat` right after `child_pid=$!` is set (after the `claude ... &` line).

5. Call `stop_heartbeat` right after `wait "$child_pid"` returns (before `child_pid=""`).

6. Add `stop_heartbeat` to the TERM trap, before killing the child:
```bash
trap 'stop_heartbeat; if [[ -n "$child_pid" ]]; then kill -TERM "$child_pid" 2>/dev/null; wait "$child_pid" 2>/dev/null; fi; exit 0' TERM
```

IMPORTANT: The `/proc/$child_pid/fd/0` approach only works on Linux. For cross-platform compatibility (tests run on macOS), use a different approach. Instead, pipe stdin from a FIFO:

Revised approach — before launching claude, create a FIFO and redirect claude's stdin from it:

Actually the simplest reliable approach: launch claude with stdin redirected from a FIFO only in telegram mode. In non-telegram modes, claude gets normal stdin.

Simplest approach that works on both macOS and Linux:

Inside the `while true` loop, BEFORE launching claude:

```bash
# Heartbeat setup (telegram mode only, per D-04, D-11, D-16)
heartbeat_pid=""
if [[ "${CLAUDE_CONNECT:-}" == "telegram" ]]; then
    HEARTBEAT_FIFO=$(mktemp -u /tmp/claude-heartbeat.XXXXXX)
    mkfifo "$HEARTBEAT_FIFO"
    # Launch claude reading from FIFO for stdin
    claude "${mode_args[@]}" "${current_args[@]}" < "$HEARTBEAT_FIFO" &
    child_pid=$!
    # Start heartbeat writer in background
    (
        # Keep FIFO open for writing (open as fd 3)
        exec 3>"$HEARTBEAT_FIFO"
        while kill -0 "$child_pid" 2>/dev/null; do
            sleep 300
            echo "" >&3 2>/dev/null || break
            echo "claude-wrapper: heartbeat sent $(date -u +%Y-%m-%dT%H:%M:%SZ)" >&2
        done
        exec 3>&- 2>/dev/null
    ) &
    heartbeat_pid=$!
else
    claude "${mode_args[@]}" "${current_args[@]}" &
    child_pid=$!
fi
```

Then after `wait "$child_pid"`:
```bash
# Stop heartbeat if running
if [[ -n "$heartbeat_pid" ]]; then
    kill "$heartbeat_pid" 2>/dev/null; wait "$heartbeat_pid" 2>/dev/null || true
    heartbeat_pid=""
fi
rm -f "${HEARTBEAT_FIFO:-}" 2>/dev/null
```

And update the TERM trap to also clean up heartbeat and FIFO:
```bash
trap 'if [[ -n "$heartbeat_pid" ]]; then kill "$heartbeat_pid" 2>/dev/null; wait "$heartbeat_pid" 2>/dev/null || true; fi; if [[ -n "$child_pid" ]]; then kill -TERM "$child_pid" 2>/dev/null; wait "$child_pid" 2>/dev/null; fi; rm -f "${HEARTBEAT_FIFO:-}" 2>/dev/null; exit 0' TERM
```

WAIT — the FIFO approach has a problem: in telegram mode, claude's stdin comes from the FIFO, not from the terminal. But in telegram mode claude doesn't read from terminal stdin anyway (it reads from the Telegram channel), so this is fine. However, for test reliability on macOS, the FIFO approach is more complex than needed.

SIMPLEST approach that works everywhere: just write to /dev/null as a "heartbeat" that logs but doesn't actually send input. The key insight is that claude in Telegram mode reads from the Telegram API, not stdin. The heartbeat's purpose (per D-06) is as a "first line of defense" — but actually reviewing D-04, it says "sends periodic newline/empty input to claude's stdin." Let's honor D-04 exactly.

Final implementation — use the FIFO approach as described above. It works on both macOS and Linux. The key modifications to `bin/claude-wrapper`:

1. Add `heartbeat_pid=""` and `HEARTBEAT_FIFO=""` at the top (after `child_pid=""` on line 33).

2. Replace the simple `claude "${mode_args[@]}" "${current_args[@]}" &` on line 42 with the conditional block above.

3. After `wait "$child_pid"` on line 44, add heartbeat cleanup.

4. Update the TERM trap on line 36 to include heartbeat cleanup.
