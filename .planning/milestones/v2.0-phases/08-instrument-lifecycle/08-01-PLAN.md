**systemd/claude-watchdog@.service** -- convert from single-instance to template:
```ini
[Unit]
Description=Claude Watchdog for %i - mode-aware forced restart

[Service]
Type=oneshot
EnvironmentFile=%h/.config/claude-restart/%i/env
ExecStart=/bin/bash -c '\
  if [ "$CLAUDE_CONNECT" = "remote-control" ]; then \
    echo "claude-watchdog[%i]: skipped restart (remote-control mode has built-in reconnection)"; \
    exit 0; \
  fi; \
  echo "claude-watchdog[%i]: restarting claude@%i (mode=$CLAUDE_CONNECT)"; \
  systemctl --user restart claude@%i.service'
```

Key changes from old `claude-watchdog.service`:
- Description includes `%i` for instance identification
- EnvironmentFile uses `%i` path (`%h/.config/claude-restart/%i/env`) instead of hardcoded `%h/.config/claude-restart/env`
- ExecStart log messages include `[%i]` for identification
- Restart target is `claude@%i.service` instead of `claude.service`

**systemd/claude-watchdog@.timer** -- convert from single-instance to template:
```ini
[Unit]
Description=Claude Watchdog Timer for %i - periodic forced restart

[Timer]
OnBootSec=8h
OnUnitActiveSec=8h
Unit=claude-watchdog@%i.service

[Install]
WantedBy=timers.target
```

Key changes from old `claude-watchdog.timer`:
- Description includes `%i`
- Timer values hardcoded to `8h` (NOT using CLAUDE_WATCHDOG_HOURS_PLACEHOLDER -- systemd timer directives cannot read env vars, per research Pitfall 2)
- Unit directive targets `claude-watchdog@%i.service` instead of `claude-watchdog.service`

Do NOT delete the old `claude-watchdog.service` and `claude-watchdog.timer` from the repo yet -- they may be needed by install.sh for reference during migration in Task 2, and removing them is a separate cleanup concern.
