### Integration Observations (non-blocking)

**1. Shell function name collision (SHEL-01)**
The `.zshrc` shell function `claude-restart` shadows the binary `claude-restart` in interactive shell. This is by design: the function is the user's launch command (starts a session via wrapper), while the binary is the in-session restart trigger (invoked from within Claude's sandbox where the zsh function isn't loaded). No functional impact.

**2. REST-03 semantic nuance**
No-arg restart writes `CLAUDE_RESTART_DEFAULT_OPTS` env var value, not the session's actual current args. Mitigated by wrapper's empty-file fallback which restores `original_args`. Both behaviors are reasonable defaults.
