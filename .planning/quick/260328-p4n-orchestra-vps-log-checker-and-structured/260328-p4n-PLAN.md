  <action>
Create `bin/claude-health` as a standalone bash script. It must work on Linux with systemd (the VPS target). Requirements:

**Usage:**
- `claude-health` -- report for ALL claude@ instances (discovered from `~/.config/claude-restart/*/`)
- `claude-health <name>` -- report for a single instance
- `claude-health --json` -- JSON output for machine consumption (orchestra)
- `claude-health --json <name>` -- JSON for single instance

**Per-instance report sections (markdown mode):**

1. **Status Summary** -- instance name, active/inactive/failed, uptime (from `systemctl show --property=ActiveEnterTimestamp`), PID, memory current (`systemctl show --property=MemoryCurrent`), memory limit (`MemoryMax` from env or 1G default)

2. **Recent Lifecycle Events** (last 24h) -- starts, stops, restarts, crashes extracted from journalctl. Use:
   ```
   journalctl --user -u "claude@${name}" --since "24 hours ago" --no-pager -o short-iso
   ```
   Filter for lines matching: "Started", "Stopped", "Main process exited", "claude exited:", "restart file found", "starting mode=", "received SIG". Show last 20 events max.

3. **Exit Codes** (last 24h) -- parse "claude exited: ..." lines from claude-wrapper log output. Show each exit with timestamp and the `describe_exit` human-readable description. Summarize: N clean exits, M signal kills, K errors.

4. **Memory Stats** -- current RSS from `MemoryCurrent` property. If `/proc/<pid>/status` is available, also show VmRSS and VmPeak.

5. **Errors/Warnings** (last 24h) -- grep journal for lines containing "error", "Error", "warn", "WARN", "fatal", "panic", "failed". Show last 10 matches. Exclude known noise (stderr-filter suppression lines).

6. **Watchdog Status** -- check `claude-watchdog@${name}.timer` state and last trigger time.

**JSON mode (`--json`):**
Output a JSON object with keys: `timestamp`, `instances` (array of objects with: `name`, `status`, `uptime_seconds`, `pid`, `memory_current_bytes`, `memory_max_bytes`, `restart_count_24h`, `error_count_24h`, `last_exit_code`, `last_exit_description`, `watchdog_active`).

**Implementation notes:**
- Use `set -euo pipefail`
- Discover instances from `$HOME/.config/claude-restart/*/` directories (same pattern as `claude-service list`)
- Use `systemctl --user show` for structured property queries (not `status` which is for humans)
- Parse journal with `--output=cat` for clean line extraction where needed
- Handle missing/stopped services gracefully (show "inactive" not crash)
- Make the script executable (chmod +x)
- Add a shebang `#!/bin/bash`

   health [--json] [instance]  Show structured health report

2. In the `case` statement at the bottom, add a `health)` branch:
   ```bash
   health)
       shift  # past 'health'
       script_dir="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
       exec "$script_dir/claude-health" "$@"
       ;;
   ```
   This passes through all remaining args (--json, instance name) to claude-health.

3. Also add "health" to the existing `start|stop|restart|...` pattern comment for clarity, but implement it as a separate case branch (not grouped with those) since it has different argument handling.
