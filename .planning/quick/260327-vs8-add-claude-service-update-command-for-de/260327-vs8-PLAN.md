Add a `deploy_skills` function to claude-service (copied from install.sh lines 33-57, identical logic: npx get-shit-done-cc@latest --global --claude, then claude plugins install superpowers@superpowers-marketplace, with warnings on failure).

Add a `do_update` function that takes a single instrument name and:

1. Resolves `script_dir` (same pattern as do_add line 52: `script_dir="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"`)
2. Resolves `work_dir` from the env file at `$CONFIG_DIR/$name/env` by grepping for `WORKING_DIRECTORY=` and extracting the value. If env dir does not exist, error "instrument '$name' not found".
3. If `$name == "orchestra"`:
   - Copy `$script_dir/../orchestra/CLAUDE.md` to `$work_dir/CLAUDE.md` (behavioral spec)
   - Write the orchestra identity hint to `$work_dir/.claude/CLAUDE.md` (same heredoc as do_add_orchestra lines 177-187, with the hardcoded "orchestra" name)
   - Echo confirmation messages
4. Else (regular instrument):
   - Copy `$script_dir/../INSTRUMENT.md.template` to `$work_dir/.claude/CLAUDE.md`
   - Run `sed_inplace "s|INSTANCE_PLACEHOLDER|$name|g" "$work_dir/.claude/CLAUDE.md"` (same as do_add line 91)
   - Echo confirmation message
5. Do NOT touch env files. Do NOT call systemctl.

Add a `do_update_all` function that:
1. Iterates `"$CONFIG_DIR"/*/` directories (same pattern as do_list line 295)
2. Calls `do_update "$name"` for each
3. After all instruments updated, calls `deploy_skills` once

Add the command dispatch in the case statement:
```
update)
    if [[ "${2:-}" == "--all" ]]; then
        do_update_all
    elif [[ -n "${2:-}" ]]; then
        do_update "$2"
        deploy_skills
    else
        echo "Usage: claude-service update <name>|--all" >&2
        exit 1
    fi
    ;;
```

Update `usage()` to add update to the Lifecycle commands section:
```
echo "  update <name>|--all  Update deployed CLAUDE.md and skills from source repo"
```

And add an example:
```
echo "  claude-service update --all           # update all instruments"
```
