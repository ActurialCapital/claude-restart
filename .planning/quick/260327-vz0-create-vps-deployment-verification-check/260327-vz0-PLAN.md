**Sections and checks to include:**

## 1. Pre-Deploy Checks
- Node.js version (node --version, expect v18+ or v20+)
- npm/npx available (npx --version)
- Claude CLI installed (claude --version)
- Git installed (git --version)
- systemd user session works (systemctl --user status, expect no errors)
- loginctl linger enabled (loginctl show-user $USER | grep Linger, expect Linger=yes)
- ANTHROPIC_API_KEY is set or ready to provide
- jq installed for config manipulation (jq --version)
- python3 available for ensure_remote_config (python3 --version)

## 2. install.sh Execution
- Clone repo to VPS (git clone)
- Run install.sh (bin/install.sh --install)
- Verify scripts installed to ~/.local/bin (ls -la ~/.local/bin/claude-wrapper claude-restart claude-service)
- Verify env.template installed (ls ~/.config/claude-restart/env.template)
- Verify systemd units installed (ls ~/.config/systemd/user/claude@.service claude-watchdog@.service claude-watchdog@.timer)
- Verify env file created with correct permissions (stat -c %a ~/.config/claude-restart/default/env, expect 600)
- Verify PATH in env file includes node binary path (grep ^PATH ~/.config/claude-restart/default/env)

## 3. Default Instance Smoke Test
- Service is enabled (systemctl --user is-enabled claude@default.service, expect "enabled")
- Service is running (systemctl --user is-active claude@default.service, expect "active")
- Service status shows no errors (systemctl --user status claude@default.service)
- Journal shows claude started (journalctl --user -u claude@default -n 20 --no-pager)
- Remote control URL available if CLAUDE_CONNECT=remote-control (look for URL in logs)

## 4. Instrument Add/Remove Lifecycle
- Add a test instrument (claude-service add test-instrument https://github.com/some/test-repo.git)
- Verify env file created (ls -la ~/.config/claude-restart/test-instrument/env)
- Verify working directory cloned (ls ~/instruments/test-instrument/)
- Verify .claude/CLAUDE.md deployed with instance name (grep "test-instrument" ~/instruments/test-instrument/.claude/CLAUDE.md)
- Verify service running (systemctl --user is-active claude@test-instrument.service)
- Verify watchdog timer active (systemctl --user is-active claude-watchdog@test-instrument.timer)
- List instruments (claude-service list, expect both default and test-instrument)
- Remove test instrument (claude-service remove test-instrument, type "yes")
- Verify cleanup: env dir gone (ls ~/.config/claude-restart/test-instrument 2>&1, expect "No such file")
- Verify cleanup: work dir gone (ls ~/instruments/test-instrument 2>&1, expect "No such file")
- Verify service stopped (systemctl --user is-active claude@test-instrument.service 2>&1, expect "inactive" or "could not be found")

## 5. Orchestra Add and Dispatch
- Add orchestra (claude-service add-orchestra)
- Verify orchestra CLAUDE.md deployed (head -5 ~/instruments/orchestra/CLAUDE.md)
- Verify orchestra identity hint (cat ~/instruments/orchestra/.claude/CLAUDE.md)
- Verify orchestra service running (systemctl --user is-active claude@orchestra.service)
- Test dispatch: send a simple prompt via claude -p from orchestra working dir (cd ~/instruments/orchestra && claude -p "echo hello" --output-format text, expect a response)
- Remove orchestra when done testing (claude-service remove --force orchestra)

## 6. Heartbeat and FIFO Verification
- For telegram mode: Check heartbeat log entries (claude-service heartbeat default, or journalctl --user -u claude@default --grep="heartbeat sent" --no-pager -n 5)
- For remote-control mode: Verify FIFO exists while service runs (ls /tmp/claude-heartbeat.* -- should see a FIFO)
- Verify FIFO cleanup after service stop: stop service, check FIFO removed (systemctl --user stop claude@default.service && ls /tmp/claude-heartbeat.* 2>&1, expect "No such file")
- Restart service after test (systemctl --user start claude@default.service)

## 7. Watchdog Timer Firing
- Verify watchdog timer is active (systemctl --user is-active claude-watchdog@default.timer, expect "active")
- Check timer schedule (systemctl --user list-timers --all | grep claude-watchdog)
- For telegram mode: manually trigger watchdog (systemctl --user start claude-watchdog@default.service) and verify restart in journal (journalctl --user -u claude-watchdog@default --no-pager -n 5)
- For remote-control mode: watchdog should skip restart (systemctl --user start claude-watchdog@default.service, then check journal for "skipped restart (remote-control mode)")

## 8. Update Command
- Run update on default (claude-service update default)
- Verify CLAUDE.md was re-deployed (check timestamp: stat ~/instruments/default/.claude/CLAUDE.md -- but default uses WORKING_DIRECTORY from env, so check accordingly)
- Run update --all (claude-service update --all)
- Verify skills re-deployed (check GSD and superpowers install output)

## 9. Restart Mechanism
- Create restart file and verify restart (claude-restart --instance default, then check journal for restart message)
- Restart with new args (claude-restart --instance default -p "test prompt", check journal)
- Verify systemctl restart works (claude-service restart default, then claude-service status default)
- Verify max restart protection: check CLAUDE_WRAPPER_MAX_RESTARTS default is 10 (grep MAX_RESTARTS in claude-wrapper or env)

## 10. Cleanup and Final State
- All expected services running (claude-service list)
- No orphaned FIFOs (ls /tmp/claude-heartbeat.* 2>&1)
- No orphaned restart files (ls ~/.config/claude-restart/*/restart 2>&1)
- Memory limit applied (systemctl --user show claude@default.service | grep MemoryMax)
