Create `.github/` and `.github/workflows/` directories, then create `deploy.yml` with:

**Triggers:**
- `push` to `main` branch
- `workflow_dispatch` (manual trigger, no inputs required)

**Job: deploy**
- Runs on `ubuntu-latest`
- Single job with a step that SSHes into the VPS and runs deployment commands

**GitHub Secrets referenced:**
- `VPS_SSH_KEY` — private SSH key for VPS access
- `VPS_HOST` — VPS hostname or IP
- `VPS_USER` — SSH username on VPS
- `VPS_REPO_PATH` — path to repo clone on VPS (default: `~/claude-restart`)

**SSH step:** Use `appleboy/ssh-action@v1` for the SSH connection. The remote script should:

```bash
cd ${VPS_REPO_PATH:-~/claude-restart}
git pull origin main
bash bin/install.sh --install
claude-service update --all
```

IMPORTANT notes for the workflow:
- `install.sh` on Linux does interactive `read` prompts (API key, working dir, connection mode). The workflow should NOT run install.sh directly because it will hang on interactive prompts. Instead, the deployment should only copy scripts and systemd units without the interactive parts. The correct approach is:
  1. `git pull origin main` to get latest code
  2. Copy scripts manually: `cp bin/claude-wrapper bin/claude-restart bin/claude-service ~/.local/bin/` and `chmod +x ~/.local/bin/claude-{wrapper,restart,service}`
  3. Copy systemd units: `cp systemd/claude@.service systemd/claude-watchdog@.service systemd/claude-watchdog@.timer ~/.config/systemd/user/` and `systemctl --user daemon-reload`
  4. Run `claude-service update --all` to re-deploy CLAUDE.md templates and skills
  5. Optionally restart services: `systemctl --user restart 'claude@*.service'` (include but comment out by default, with a note that uncommenting will restart active Claude sessions)

- Add a concise comment at the top of the workflow explaining what it does and what secrets are required
- Use `|` (literal block scalar) for the multi-line script in the ssh-action step
