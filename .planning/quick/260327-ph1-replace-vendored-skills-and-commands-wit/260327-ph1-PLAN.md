1. Rewrite the `deploy_skills()` function in `bin/install.sh` (lines 84-106) to:

```bash
deploy_skills() {
    local claude_dir="$HOME/.claude"
    local gsd_repo="https://github.com/gsd-build/get-shit-done.git"
    local superpowers_repo="https://github.com/obra/superpowers.git"

    # Deploy GSD skills (get-shit-done)
    if [[ -d "$claude_dir/get-shit-done/.git" ]]; then
        git -C "$claude_dir/get-shit-done" pull --ff-only 2>/dev/null || echo "Warning: GSD skills pull failed (using existing)"
        echo "Updated GSD skills in $claude_dir/get-shit-done/"
    else
        rm -rf "$claude_dir/get-shit-done"
        if git clone "$gsd_repo" "$claude_dir/get-shit-done" 2>/dev/null; then
            echo "Cloned GSD skills to $claude_dir/get-shit-done/"
        else
            echo "Warning: Failed to clone GSD skills from $gsd_repo (skipping)"
        fi
    fi

    # Deploy superpowers commands
    if [[ -d "$claude_dir/commands/.git" ]]; then
        git -C "$claude_dir/commands" pull --ff-only 2>/dev/null || echo "Warning: superpowers commands pull failed (using existing)"
        echo "Updated superpowers commands in $claude_dir/commands/"
    else
        rm -rf "$claude_dir/commands"
        if git clone "$superpowers_repo" "$claude_dir/commands" 2>/dev/null; then
            echo "Cloned superpowers commands to $claude_dir/commands/"
        else
            echo "Warning: Failed to clone superpowers commands from $superpowers_repo (skipping)"
        fi
    fi
}
```

Key design decisions:
- Remove the `skills_src` and `commands_src` local variables entirely -- no more referencing local vendored dirs.
- Use `git clone` for fresh installs, `git pull --ff-only` for updates.
- Check for `.git` directory to detect existing clones vs. old cp-based deployments.
- `rm -rf` before clone handles the case where dir exists but is not a git repo (leftover from old cp-based deploy).
- Keep `--ff-only` to avoid merge conflicts during pull.
- Graceful fallback: if clone/pull fails, warn and continue (network may be unavailable).
